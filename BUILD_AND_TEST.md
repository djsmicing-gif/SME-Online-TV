# Build and Test Guide for SME Online TV\n\n## 📋 Complete Workflow\n\nThis guide covers all build and test operations for the SME Online TV Android project.\n\n---\n\n## 🔧 Prerequisites\n\n### System Requirements\n- **OS**: Windows, macOS, or Linux\n- **Java**: JDK 11 or later\n- **Android SDK**: 34 (API Level 34)\n- **Min SDK**: 24 (API Level 24)\n- **RAM**: 8GB minimum (16GB recommended)\n- **Storage**: 15GB free space\n\n### Software Installation\n\n#### 1. Install Java/JDK 11+\n```bash\n# macOS (via Homebrew)\nbrew install java11\n\n# Windows (via Chocolatey)\nchoco install jdk11\n\n# Linux (Ubuntu/Debian)\nsudo apt-get install openjdk-11-jdk\n```\n\n#### 2. Install Android Studio\n- Download from https://developer.android.com/studio\n- Follow installation instructions for your OS\n\n#### 3. Set Environment Variables\n```bash\n# macOS/Linux - Add to ~/.bashrc or ~/.zshrc\nexport ANDROID_HOME=$HOME/Library/Android/sdk\nexport PATH=$PATH:$ANDROID_HOME/tools\nexport PATH=$PATH:$ANDROID_HOME/tools/bin\nexport PATH=$PATH:$ANDROID_HOME/platform-tools\n\n# Windows - Set in System Environment Variables\nANDROID_HOME=C:\\Users\\YourUsername\\AppData\\Local\\Android\\sdk\nPath=%Path%;%ANDROID_HOME%\\tools;%ANDROID_HOME%\\platform-tools\n```\n\n#### 4. Install Android SDK Components\n```bash\n# Using Android Studio\n# Settings > Appearance & Behavior > System Settings > Android SDK\n# Install:\n# - Android SDK 34 (Android 14)\n# - Android SDK 24 (Android 7.0)\n# - Google Play Services\n# - Android Emulator\n# - Android SDK Tools\n```\n\n---\n\n## 🏗️ Build Operations\n\n### 1. Clean Build\nRemoves all build artifacts and rebuilds from scratch.\n\n```bash\n# Navigate to project root\ncd SME-Online-TV\n\n# Clean\n./gradlew clean\n\n# Build\n./gradlew build\n```\n\n**Output:**\n```\n✓ Cleans build directory\n✓ Removes intermediate files\n✓ Compiles fresh code\n✓ Runs code analysis\n✓ Creates build artifacts\n```\n\n### 2. Debug Build (Development)\nBuilds with debugging symbols and optimizations disabled.\n\n```bash\n./gradlew assembleDebug\n```\n\n**Output:**\n```\nBuild outputs:\n- app/build/outputs/apk/debug/app-debug.apk (Development APK)\n```\n\n**Features:**\n- Full debugging symbols\n- Logcat output\n- Breakpoint support\n- No optimizations\n- Faster builds\n\n### 3. Release Build (Production)\nBuilds optimized version with ProGuard/R8 minification.\n\n```bash\n./gradlew assembleRelease\n```\n\n**Output:**\n```\nBuild outputs:\n- app/build/outputs/apk/release/app-release-unsigned.apk\n```\n\n**Features:**\n- Code minification (R8/ProGuard)\n- Resource shrinking\n- Optimizations enabled\n- Production-ready\n- Requires signing\n\n### 4. Bundle Build (Play Store)\nCreates Android App Bundle for Google Play Store.\n\n```bash\n./gradlew bundleRelease\n```\n\n**Output:**\n```\nBuild outputs:\n- app/build/outputs/bundle/release/app-release.aab\n```\n\n**Features:**\n- Dynamic delivery\n- Optimized per device\n- Smaller downloads\n- Play Store compatible\n\n### 5. Build Variants\n\n```bash\n# List all build variants\n./gradlew tasks | grep build\n\n# Build specific variant\n./gradlew assembleBuildVariantNameDebug\n./gradlew assembleBuildVariantNameRelease\n\n# Example: Build with custom flavor\n./gradlew assembleProductionDebug\n./gradlew assembleProductionRelease\n```\n\n### 6. View Build Report\n\n```bash\n# HTML build report\n./gradlew build\n# Open: app/build/reports/build/index.html\n\n# Build scan\n./gradlew build --scan\n# Opens https://scans.gradle.com with detailed report\n```\n\n---\n\n## 🧪 Testing Operations\n\n### 1. Unit Tests\nTests for individual components (local JVM).\n\n```bash\n# Run all unit tests\n./gradlew test\n\n# Run specific test class\n./gradlew test --tests com.smeonlinetv.app.util.DateUtilTest\n\n# Run specific test method\n./gradlew test --tests com.smeonlinetv.app.util.DateUtilTest.testFormatDate\n\n# Run with debug output\n./gradlew test --debug\n\n# Run tests with coverage\n./gradlew test jacocoTestReport\n```\n\n**Output Locations:**\n```\napp/build/test-results/test/index.html (Report)\napp/build/reports/jacoco/test/html/index.html (Coverage)\n```\n\n### 2. Instrumentation Tests\nTests running on Android device/emulator.\n\n```bash\n# Start emulator first\n./gradlew connectedAndroidTest\n\n# Run specific instrumented test\n./gradlew connectedAndroidTest --tests com.smeonlinetv.app.ui.screens.*\n\n# With debug output\n./gradlew connectedAndroidTest -i\n\n# Generate report\n./gradlew connectedAndroidTest --continue\n```\n\n**Requirements:**\n- Android emulator running or device connected\n- USB debugging enabled\n- Proper permissions granted\n\n**Output Locations:**\n```\napp/build/reports/androidTests/connected/index.html\napp/build/test-results/connected/\n```\n\n### 3. Code Analysis\n\n#### Lint Analysis\n```bash\n# Run lint checks\n./gradlew lint\n\n# Run specific lint checks\n./gradlew lint -q\n\n# Generate HTML report\n./gradlew lintDebug\n```\n\n**Output:**\n```\napp/build/reports/lint-results-debug.html\n```\n\n#### Static Analysis Tools\n```bash\n# With ktlint (Kotlin linting)\n./gradlew ktlint\n\n# With detekt (Kotlin code quality)\n./gradlew detekt\n\n# Generate all reports\n./gradlew check\n```\n\n### 4. Code Coverage\n\n```bash\n# Generate coverage report for unit tests\n./gradlew test jacocoTestReport\n\n# Generate coverage report for instrumented tests\n./gradlew connectedAndroidTest jacocoTestReport\n\n# Combined coverage\n./gradlew jacocoTestReport\n\n# View coverage\n# Open: app/build/reports/jacoco/test/html/index.html\n```\n\n**Coverage Targets:**\n- Line coverage: > 80%\n- Branch coverage: > 75%\n- Method coverage: > 85%\n\n### 5. Performance Testing\n\n```bash\n# Profile build time\n./gradlew build --profile\n# Open: build/reports/profile/ for details\n\n# Benchmark tests\n./gradlew connectedBenchmarkTest\n\n# Memory profiling\n./gradlew test --jvmArgs=\"-Xmx2g\"\n```\n\n### 6. Test Reports\n\n```bash\n# Generate comprehensive test report\n./gradlew test --info\n\n# View HTML reports\napp/build/reports/tests/test/index.html  # Unit tests\napp/build/reports/androidTests/connected/ # Instrumented tests\n```\n\n---\n\n## 📦 Installation Operations\n\n### 1. Install Debug APK\n\n```bash\n# Build and install\n./gradlew installDebug\n\n# Just install existing APK\nadb install app/build/outputs/apk/debug/app-debug.apk\n\n# Install and run\n./gradlew installDebug\nadb shell am start -n com.smeonlinetv.app/.MainActivity\n```\n\n### 2. Install Release APK\n\n```bash\n# First sign the APK\njarsigner -verbose -sigalg SHA1withRSA -digestalg SHA1 \\\n  -keystore my-release-key.keystore \\\n  app/build/outputs/apk/release/app-release-unsigned.apk \\\n  alias_name\n\n# Verify signing\njarsigner -verify app/build/outputs/apk/release/app-release.apk\n\n# Install signed APK\nadb install app/build/outputs/apk/release/app-release.apk\n```\n\n### 3. Check Installation\n\n```bash\n# List installed packages\nadb shell pm list packages | grep smeonlinetv\n\n# Check app info\nadb shell dumpsys package com.smeonlinetv.app\n\n# View installed app version\nadb shell pm dump com.smeonlinetv.app | grep versionName\n```\n\n### 4. Uninstall App\n\n```bash\n# Uninstall app\nadb uninstall com.smeonlinetv.app\n\n# Uninstall and keep data\nadb uninstall -k com.smeonlinetv.app\n```\n\n---\n\n## 🔍 Dependency Management\n\n### 1. View Dependencies\n\n```bash\n# List all dependencies\n./gradlew dependencies\n\n# Tree format\n./gradlew dependencies --configuration debugRuntimeClasspath\n\n# Check for updates\n./gradlew dependencyUpdates\n\n# Generate dependency report\n./gradlew htmlDependencyReport\n# Open: build/reports/project/dependencies.html\n```\n\n### 2. Update Dependencies\n\n```bash\n# Check for available updates\n./gradlew dependencyUpdates\n\n# Update to latest versions\n# Manually edit app/build.gradle.kts\n\n# Sync after updates\n./gradlew sync\n```\n\n### 3. Dependency Conflicts\n\n```bash\n# Detect conflicts\n./gradlew dependencies --configuration debugRuntimeClasspath | grep -i conflict\n\n# Force specific version\n# Add to build.gradle.kts:\n# configurations.all {\n#     resolutionStrategy {\n#         force 'library:version'\n#     }\n# }\n```\n\n---\n\n## 📊 Advanced Build Commands\n\n### 1. Build Caching\n\n```bash\n# Enable build cache\n./gradlew build --build-cache\n\n# View cache info\n./gradlew tasks --all | grep cache\n\n# Clean cache\nrm -rf .gradle/build-cache\n```\n\n### 3. clean\n\n# Solution 2: Update Gradle wrapper\n./gradlew wrapper --gradle-version=8.1.2\n\n# Solution 3: Invalidate Android Studio cache\n# Android Studio > File > Invalidate Caches > Clear Files and Restart\n```\n\n**Issue: \"Build timeout\"**\n```bash\n# Increase timeout in gradle.properties\norg.gradle.timeout=120000\n\n# Or run with more memory\n./gradlew build -Xmx4g\n```\n\n**Issue: \"Out of memory\"**\n```bash\n# Increase heap size\nexport GRADLE_OPTS=\"-Xmx2048m\"\n./gradlew build\n```\n\n### Test Issues\n\n**Issue: \"Tests fail randomly\"**\n```bash\n# Run tests sequentially\n./gradlew test --no-parallel\n\n# Run with verbose output\n./gradlew test --info\n```\n\n**Issue: \"Instrumented tests won't run\"**\n```bash\n# Ensure emulator is running\nemulator -avd YourAVDName\n\n# Check connected devices\nadb devices\n\n# Clear app data before test\nadb shell pm clear com.smeonlinetv.app\n```\n\n---\n\n## 📚 Best Practices\n\n### Build Optimization\n- ✅ Use incremental builds\n- ✅
